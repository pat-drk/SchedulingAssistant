Teams Shifts Assistant

A localized scheduling engine designed for high-precision staff coordination and seamless Microsoft Teams integration.

The Teams Shifts Scheduling Assistant is a specialized browser application for managing staff assignments across multiple time segments. By utilizing local SQLite technology, your data remains secure and private on your workstation.

5. Database Management

Creating & Opening Databases

The application uses local SQLite databases stored as .db files on your machine. All data remains on your workstation‚Äînothing is uploaded to the cloud.

Action Description When to Use
New DB Creates a fresh database in memory with initial schema. Starting a new scheduling system or semester.
Open DB Loads an existing .db file for editing. Regular daily use to modify schedules.

Saving Data

Changes are held in memory until you explicitly save them.

Save (Ctrl+S)

Writes all changes back to the currently opened file and syncs with others.

  • ‚Ä¢ Requires write permission on the file
  • ‚Ä¢ Syncs changes to the shared changes folder
  • ‚Ä¢ Use frequently to avoid data loss

Save As

Creates a new copy of the database at a different location.

  • ‚Ä¢ Use to create backups
  • ‚Ä¢ Prompts for new file location
  • ‚Ä¢ Creates a separate sync environment

Multi-User Sync System

Multiple users can edit the database simultaneously. The app uses an operational log system with automatic conflict resolution to merge changes seamlessly.

Collaborative Editing

When opening a database, you'll be prompted to enter your Work Email to identify your changes. All users can edit simultaneously‚Äîno more waiting for locks!

How the Sync System Works
  1. 1. User Identification: When you open a file, enter your email to identify your changes.
  2. 2. Change Tracking: All database operations are tracked with timestamps and user info.
  3. 3. Auto-Merge: Non-conflicting changes from other users are merged automatically in the background (every 30 seconds).
  4. 4. Conflict Resolution: If conflicts are detected (same field edited by multiple users), a dialog helps you resolve them.
  5. 5. Sync Status: The sync indicator in the top bar shows current sync state and active users.

Automatic Migrations

Older database files are automatically upgraded to the latest schema when opened. This ensures compatibility across application versions.

Feature Details
Backward Compatible Old databases open seamlessly in new versions.
Schema Updates Migrations add new tables/columns without data loss.
Zero User Action Migration happens automatically on database open.
No Downgrade Once migrated, file may not work in older app versions.

Best Practice

Always make a backup copy (Save As) before major schema migrations, especially if working with critical historical data.

Training & Qualifications

The Training page has two main functions: tracking role qualifications for all staff, and managing trainees in their first 6 months with rotation tracking across key areas.

Trainee Dashboard

New employees in their first 6 months (based on their Start Date) are tracked as trainees. The dashboard helps ensure each trainee is exposed to all required training areas: Dining Room, Machine Room, Veggie Room, and Receiving.

Training Status Cards

Each trainee has a status card showing their start date, time remaining, progress bar, and checklist of completed/pending training areas.

Training Progress Report

Export an HTML summary of all trainees with their completion status for each training area and overall progress percentage.

Training Alerts

Prominent warnings for at-risk trainees: those approaching their 6-month mark without completing all areas, or those who haven't rotated in 2+ months.

Suggested Assignments

Intelligent recommendations for which area each trainee should be assigned to next, prioritizing areas they haven't experienced.

7. Admin Configuration

The Admin tab provides access to all system configuration components. These settings define the structure of your scheduling system.

Time-Off Manager

Import and manage employee time-off entries. The system prevents assignments during time-off periods and automatically splits shifts around time-off blocks during export.

Import from XLSX

Upload a CSV/XLSX file exported from Microsoft Teams with time-off records. Format: Person Email, Start Date/Time, End Date/Time.

Manual Add

Create individual time-off entries manually. Select person, date range, and times. Useful for ad-hoc requests.

View/Delete Entries

Browse all time-off records. Delete entries that are no longer relevant or were imported in error.

Department Events

Schedule department-wide events like meetings, training sessions, or gatherings that automatically assign all active crew members. Events display as banners on the Daily Run Board and are included in daily Excel exports.

Field Description
Title Event name (e.g., "Department Meeting", "Safety Training"). Used as the segment name for assignments.
Date The date the event takes place.
Start Time / End Time Event duration. Regular segments that overlap with this time will be automatically shortened or split.
Group Optional: Assign to a group for Excel export categorization. If not set, uses the event title as the group name.
Role Optional: Assign a specific role for the event. Defaults to the selected group name if not specified.
Description Optional notes about the event (e.g., location, agenda topics).
Auto-Assignment

When you create an event, all active crew members are automatically assigned. The event title becomes the segment name for these assignments.

Segment Adjustment

Events that overlap with regular segments (AM, PM, Lunch) automatically shorten or split those segments. A banner on the Daily Run Board shows affected segments.

Excel Export

Daily schedule exports include an "Events" sheet listing all department events with their attendees, making it easy to track participation.

Daily Run Board Banner

Events display as informational banners at the top of the Daily Run Board, showing event time, title, and any affected segments.

Segment Adjustments

Create dynamic time adjustments for segments based on one or more conditions. Segment adjustments now support multiple conditions with AND/OR logic for complex scheduling scenarios.

Field Purpose
Conditions One or more segment+role conditions that must be met. Click "Add Condition" to create multi-condition rules.
Logic Operator For multi-condition rules: AND (all conditions must be met) or OR (any condition can trigger).
Condition Segment The segment that triggers the adjustment (must be assigned on that date).
Condition Role Optional: Require a specific role in the condition segment. Leave as "Any" to match any role.
Target Segment Which segment's time to adjust when conditions are met.
Adjust Start/End Whether to modify the target segment's start or end time.
Baseline Reference point for the adjustment (condition start/end or target start/end).
Offset Minutes Number of minutes to add/subtract from baseline (positive or negative).
Preview Visual timeline showing how the adjustment affects segment times.
Example 1: Single Condition - Link AM to Lunch
  • Condition: Lunch (Any role)
  • Target Segment: AM
  • Adjust: End Time
  • Baseline: condition.start (Lunch's start time)
  • Offset: 0 minutes
  • Result: AM's end time dynamically matches Lunch's start time on days when Lunch is assigned.
Example 2: Multi-Condition with AND - Complex Lunch Rules
  • Conditions: Lunch (Any role) AND AM (Specific Manager Role)
  • Logic: AND (both conditions must be met)
  • Target Segment: PM
  • Adjust: Start Time
  • Baseline: condition.end (Lunch's end time)
  • Offset: 60 minutes
  • Result: PM start is delayed by 60 minutes after Lunch ends, but ONLY when both Lunch is scheduled AND a specific manager is in AM.
Example 3: Multi-Condition with OR - Fallback Scenarios
  • Conditions: Early (Any role) OR Extended-Early (Any role)
  • Logic: OR (any condition can trigger)
  • Target Segment: AM
  • Adjust: Start Time
  • Baseline: condition.end (Early/Extended-Early end time)
  • Offset: 0 minutes
  • Result: AM start adjusts to match whichever early segment is scheduled (either Early OR Extended-Early).

10. Troubleshooting

"No database loaded"

Click New DB or Open DB in the top bar to activate the system.

Changes not saving

Verify write permissions for your .db file and the changes folder. Ensure the file location is accessible.

Sync conflicts appearing

When multiple users edit the same field, a conflict dialog will appear. Choose "Keep Yours" to keep your changes, "Keep Theirs" to accept the other user's changes, or "Keep Both" where applicable.

Trainee not appearing in training dashboard

Ensure the person has a Start Date set in their profile. Only people within their first 6 months from their start date appear as trainees.

Email prompt keeps appearing

The app prompts for your email each time you open a file. This identifies your changes in the sync system and is expected behavior for multi-user collaboration.
